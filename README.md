# Friday Night Funkin' - Fawn Engine (working title)
<sub>Uh oh! Your tryin to kiss ur hot girlfri-... We're doing this again?</sub>

Fawn Engine is an engine for Friday Night Funkin' which uses **none** of the original game's code. 

<sub>Can we even call this an engine?... A reimplementation? We'll figure that out later.</sub>

Fawn Engine is written from the ground up in C++ with SDL2, using Friday Night Funkin's official code as reference/blueprints.

The goal of this engine is to study the code of FNF (specifically the V-Slice update!), try to understand it, and rebuild in C++.

I aim for this project to look and feel as close to V-Slice as possible, as well as possible legacy hardware ports in the future!

NOTE: This engine is VERY early in development (like.. SUPER early). Source code as well as a build will be public soon, stay tuned!

# Roadmap
### The basic stuff!
- [x] Barebones gameplay + inputs
- [x] Audio loading
- [x] Chart loading (metadata AND charts)
- [ ] BF vocals muting on miss
- [ ] Menus accurate to V-Slice (or... ANY menus...)

### The Other Cool Stuff™
- [ ] Accessibility settings (adjustable screen shaking, panning, flashing lights etc)
- [ ] Mod support! This engine aims to support mods from the big three engines for modding (V-Slice, Psych, and Codename).
- [ ] Support for remix/conversion mods (stuff like B-Side and B3!!)
- [ ] Customizable menus with downloadable themes

### Long term goals!!
- [ ] In-game mod downloading, prioritizing GameBanana
- [ ] Mobile builds (Android first with the possibility of ios later)
- [ ] Builds for legacy devices/consoles
- [ ] ...Online multiplayer? (don't count on it...)
